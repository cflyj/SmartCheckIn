# 毕业设计中期报告撰写指南（以 SmartCheckIn 为例）

> 适用：将「移动端 Web 智能签到系统」类全栈项目作为毕业设计时，撰写**中期报告**或准备**中期答辩**的参考提纲。  
> 可与 `docs/PROJECT_ENGINEERING_RETROSPECTIVE.md`（工程复盘）对照使用：前者偏**论文与答辩表述**，后者偏**技术细节**。

---

## 一、中期报告要回答的三个问题

评委通常快速确认：

1. **做什么**：一句话题目 + 明确范围（做到哪、明确不做什么）。
2. **做到哪了**：已完成模块、能否演示、进度是否可信。
3. **后面怎么收尾**：剩余工作、风险与时间安排。

写作时每一段都尽量能对应到上述三点之一。

---

## 二、建议目录结构（正文约 8～15 页较常见）

### 第一章 绪论

- **选题背景**：课堂签到、活动考勤、防代签、可追溯等现实需求。
- **意义**：提高效率、数据可查、结合地理位置与动态二维码等应用价值。
- **本文主要工作概述**：需求分析、总体设计、前后端实现、测试与部署等（按个人实际分工写）。

### 第二章 需求分析

- **用户角色**：组织者、参与者；若有组织、管理员等角色一并列出。
- **功能需求**：活动管理、地理签到、二维码签到、签到记录与统计等，可用列表或简单用例表。
- **非功能需求**：移动端适配、安全上下文（HTTPS）与敏感 API、数据持久化、可部署性等（点到即可，不必过长）。

### 第三章 总体设计

- **系统架构**：浏览器 → 前端 SPA → 后端 API → 数据库（建议配**结构框图**）。
- **技术选型**：Vue 3 + Vite、Express、SQLite、JWT 等，每项用一两句话说明选型理由（成熟、易部署、适合课题规模）。
- **关键设计要点**（概念级即可）：签到时间窗、地理围栏判定思路、二维码令牌与防重放、统一错误码与鉴权等。

### 第四章 已完成工作与阶段性成果（中期重点）

建议用**表格 + 完成度**呈现，例如：

| 模块           | 状态        | 简要说明           |
|----------------|-------------|--------------------|
| 用户注册登录、JWT | 已完成      | …                  |
| 活动创建与管理   | 已完成      | …                  |
| 地理签到         | 已完成/优化中 | …                |
| 二维码签到       | 已完成      | …                  |
| 组织与邀请码     | 已完成      | …                  |
| 部署与联调文档   | 已完成      | …                  |
| 测试用例/论文初稿 | 进行中      | …                  |

- 附 **2～3 张运行截图**（登录、组织者端、参与者签到成功或统计页），注意**脱敏**。
- 若有 Git 提交记录或版本说明，可简要写「代码已持续迭代至某版本」。

### 第五章 已遇到的问题与解决措施

选 **3～5 条** 有技术含量的即可，每条建议固定格式：**现象 → 原因 → 解决办法**。

可与本项目相关的典型条目（按实际是否实现勾选）：

- 开发环境跨域：通过 Vite 将 `/api` 代理到后端，避免浏览器 CORS。
- 生产环境 SPA 刷新 404：Express（或 Nginx）对非 API 路径回退到 `index.html`。
- 定位、摄像头依赖 HTTPS：说明测试环境（如正式域名、或临时 HTTPS 隧道）与注意事项。
- 部分国产浏览器限制剪贴板或摄像头：降级方案（如 `execCommand` 复制、只读输入框、拍照识码/手动输入令牌）与用户提示文案。
- 扫码结果为完整 URL 而非纯 token：前后端统一从查询参数解析 `token`，保证兼容。

### 第六章 后期工作计划

- **待完成项**：例如补充测试、撰写论文第 × 章、答辩 PPT、可选 UI/无障碍优化等。
- **时间安排**：建议用**周次或月份**的简易甘特表，写清「何时完成何项」，便于导师把关。

### 参考文献

- 数量一般 **8～15 篇**，含 Vue/Express 等官方或权威文档、Web 安全与 HTTPS 相关资料，以及 1～2 篇与签到、LBS 相关的中文文献即可。
- 格式按学校要求（如 GB/T 7714）统一。

---

## 三、写作与答辩加分细节

- **进度表述诚实**：写「核心功能已基本实现，测试与论文撰写进行中」比夸大「全部完成」更安全。
- **突出个人工作**：若是小组课题，明确本人负责前端/后端/部署中的具体模块。
- **中期答辩 PPT**：约 10～15 页为宜，**架构一图、进度一表、演示一屏**，正文少字、多图多表。

---

## 四、与学校模板的关系

若学校下发**固定封面、固定章节标题**，将本文**第二节各章内容**映射进模板的一、二、三级标题即可；通常**第四章（已完成工作）**与**第六章（后期计划）**是中期考核中最受关注的部分。
